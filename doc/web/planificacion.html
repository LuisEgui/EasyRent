Organización del trabajo

Github

Tenemos un repositorio de github donde repartiremos las tareas y subiremos el código. Al empezar a trabajar en cada entrega, pensaremos cuales son las tareas que hay que realizar para completar cada hito y crearemos 'issues' que representan esas tareas.

Según la catidad y la complejidad de 'issues' en cada tarea, definiremos cuantas personas se necesitan en cada uno y los asignaremos equitativamente.

El trabajo sigue cuatro etapas representadas en entregas. Las comentamos a continuación, con los 'issues' seguidos en la primera entrega.
